# Graph-based Validation System with LLM

A graph-based validation system that retrieves attributes and uses LLM assistance to infer dependencies between them. This application validates complex data structures and generates rule-based dependency graphs to ensure data consistency and integrity.

## Running the Project Locally

### Prerequisites

- Python 3.11 or higher: [python.org](https://www.python.org)
- pip: Usually comes with Python, but can be installed separately if needed: `py -m ensurepip --upgrade`
- Git & GitHub: For version control

### Installation & Setup

1. Create and activate a virtual environment:

   ```
   python -m venv .venv
   .venv\Scripts\activate
   ```

2. Install dependencies:

   ```
   pip install -r requirements.txt
   ```

3. Set up environment variables:

   ```
   cp env.example .env
   ```

   Edit the `.env` file with your configuration (API keys, database URLs, etc.)

4. Start the FastAPI development server:

   ```
   uvicorn app.main:app --reload --reload-dir app
   ```

   The development server will launch at http://127.0.0.1:8000

   Access the interactive API documentation at: http://127.0.0.1:8000/docs

### Running Tests

To run the test suite:

```
pytest
```

## Running with Docker

### Prerequisites

- Docker: [docker.com](https://www.docker.com)
- A configured `.env` file (see [Installation & Setup](#installation--setup))

### Build the Image

```
docker build -t graph-validation .
```

### Run the Container

**Important:** The `.env` file is not included in the Docker image. You need to pass environment variables at runtime.

Run in the foreground:

```
docker run --rm -p 8000:8000 --env-file .env graph-validation
```

Or run in the background (detached):

```
docker run -d -p 8000:8000 --env-file .env graph-validation
```

Alternatively, pass individual environment variables:

```
docker run --rm -p 8000:8000 -e OPENAI_API_KEY=your_key_here graph-validation
```

Access the API at: http://127.0.0.1:8000/docs
